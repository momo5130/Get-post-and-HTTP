HTTP1.1

  • Introduced in 1997.
  • Internet ecosystem was changing at a rapid pase.
  • During that duration more search engines and websites were being created with heavy pages having more images and data, and also the resource usage increased.
  • In order to cope-up with that, more features were required.
  • Addition to the fatures of HTTP1, HTTP1.1 came up with features like CORS, Keep-alive to make internet fast.

Flaws in HTTP1.1

  • Header information is repeated in every request, which cannot be compressed, even though the data can be compressed.
  • More focused on gzip, minifying CSS/JS, caching etc.
  • TCP connection gets blocked till the response is received.
  • HTTP1.1 has 6 TCP connection which were still not enough for the clients.
  • Keep-alive option enables re-using of the same TCP connection for multiple HTTP request.

HTTP2.0

  • HTTP2.0 came in 2015
  • HTTP2.0 is build on top of HTTP1.1, which includes all the features of HTTP1.1.

Features of HTTP2.0

  • In HTTP2.0 single secured TCP connection is being used, in which streams are created in which every HTTP request is sent as a stream or stream frame. So single TCP connection pipeline is used and in that it is compulsory to have TLS setup, which makes it faster. We don't have to create mutiple TCP connections.
  • In HTTP2, HPACK was added, which separates the Header data from the request and can be zipped.
  • HPACK enables automatic caching of Header data which reduces HTTP request size
  • PUSH frames were also included in HTTP2.0 which enables the client to send mandatory resources in advance along with an HTTP response.
